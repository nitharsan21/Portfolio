GsbFrais - Logiciel web

Contexte

L’entreprise GSB est issue de la fusion de l’entreprise pharmaceutique américaine Galaxy et du conglomérat européen Swiss Bourdin.

Elle organise des visites dans les laboratoires afin de promouvoir leurs recherche pharmaceutique. Lors de ces visites cette organisation doit prendre en charge les frais des visiteurs (transport, restauration, logement...).

Le laboratoire GSB souhaite migrer son application de gestion de frais des visiteurs vers Symfony4. Cette application a été tout d'abord développée sans s'appuyer sur une architecture MVC. L'application, dans son état actuel, ne fonctionne pas. Le système (l’application) doit répondre aux sollicitations de l’utilisateur.

Environnement technologique

Généralités :
  • L'application a été développé dans un environnement Linux.
  • L'application web dédiée à la saisie, gestion et consultation des frais sera développée avec Symfony 3.4.
Langages utilisés :
  • HTML.
  • CSS.
  • PHP.
  • TWIG.
  • Annotations.
  • YAML.
  • SQL (DQL/MySQL).
Outils :
  • Architecture MVC de symfony 4 composé de :
    • Un outil de manipulation de bases de données Doctrine
    • Un outil de manipulation de php dans le code HTML Twig
    • Un outil de gestion des routes Composer
    • Un outil d'installation du framework Installer
  • Serveur LAMP (Linux, Apache2, MySQL5, PHP7)
  • IDE netbeans

Fonctionnalités de l’application

  • Authentification du visiteur
    • Navigation
      • Saisir ou modifier la fiche de frais du mois
      • Visualiser la fiche du mois
      • Visualiser l'historique des fiches
      • Modifier les paramètre de compte
      • Déconnexion
